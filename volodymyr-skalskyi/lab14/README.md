## Хід роботи

### Загальні принципи проєктування

#### DRY (Don't Repeat Yourself) - Не повторюйся

Принцип, який полягає в тому, що кожна частина знання повинна мати єдине, однозначне, авторитетне представлення в системі. Код не повинен дублюватися.

#### AHA (Avoid Hasty Abstractions) - Уникай поспішних абстракцій

Принцип, який застерігає від створення абстракцій занадто рано, до того, як стане зрозуміло, які саме абстракції дійсно потрібні.

#### KISS (Keep It Simple, Stupid) - Тримай це простим

Принцип, який наголошує на простоті дизайну та униканні непотрібної складності.

### Комплекс принципів SOLID

SOLID - це акронім, який позначає п'ять принципів об'єктно-орієнтованого програмування та проєктування:

1. **S - Single Responsibility Principle (Принцип єдиної відповідальності)**

   - Клас повинен мати лише одну причину для зміни
   - Кожен клас повинен відповідати за одну функціональність

2. **O - Open/Closed Principle (Принцип відкритості/закритості)**

   - Програмні сутності повинні бути відкриті для розширення, але закриті для модифікації

3. **L - Liskov Substitution Principle (Принцип підстановки Лісков)**

   - Об'єкти в програмі повинні бути замінними на екземпляри їх підтипів без порушення роботи програми

4. **I - Interface Segregation Principle (Принцип розділення інтерфейсу)**

   - Клієнти не повинні залежати від інтерфейсів, які вони не використовують

5. **D - Dependency Inversion Principle (Принцип інверсії залежностей)**
   - Модулі високого рівня не повинні залежати від модулів низького рівня. Обидва повинні залежати від абстракцій

### Принцип єдиної відповідальності (SRP)

Принцип єдиної відповідальності стверджує, що кожен клас повинен мати лише одну причину для зміни. Це означає, що клас повинен виконувати лише одну функцію або мати лише одну відповідальність.

**Переваги дотримання SRP:**

- Легше тестувати код
- Простіше підтримувати та змінювати
- Краща читабельність коду
- Менша залежність між компонентами

**Проблеми порушення SRP:**

- Складність тестування
- Складність підтримки
- Вищі ризики при внесенні змін
- Порушення принципу розділення відповідальностей
