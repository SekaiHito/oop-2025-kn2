## **Хід роботи**

Поняття рефакторингу

Рефакторинг - це процес покращення структури програмного коду без зміни зовнішньої поведінки програми. Його мета полягає не у додаванні нового функціоналу, а в упорядкуванні вже наявного, щоб він став більш зрозумілим, простим для підтримки та зручним для розширення.

3. **Основні техніки рефакторингу**  

    Розглянемо кілька основних технік рефакторингу:
    Перша з яких Extract Method суть полягає у розбитті довгих функцій, методів на кілька менших:

    ```python

    def do_order(order):
        total = 0
        for item in order.items:
            total += item.price * item.quantity

        print("Total:", total)
        file = open("log.txt", "a")
        file.write(f"Order total: {total}\n")
        file.close()
    
    #Extract Method

    def calculate_total(order):
        return sum(item.price * item.qty for item in order.items)

    def log_total(total):
        with open("log.txt", "a") as f:
            f.write(f"Order total: {total}\n")

    def process_order(order):
        total = calculate_total(order)
        print("Total:", total)
        log_total(total)

    ```

    Rename - покращення назв так щоб її імя описувало її роботу

    ```python
    def calc(a):
    return a * 3.14

    #Rename

    def calculate_circle_area(radius):
    return radius * 3.14

    ```

    Replace Magic Number - заміна магічних чисел на константу

    ```python
    if sec > 100:
    print("Time!")

    #Replace

    MAX_SEC = 100
    if sec > MAX_SEC:
        print("Time!")

    ```

    Тобто в основі любого методу буде покращення розуміння коду що дає можливість просто підтримувати та зручно розширювати.
