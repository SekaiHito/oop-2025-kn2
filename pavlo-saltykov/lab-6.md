## **Хід роботи**

Завдання полягає у створенні невеликої програми з простими функціями, які будуть наочно демонструвати принципи тестування коду. Для цих функцій потрібно написати тести, що перевірятимуть їхню правильність та коректність роботи. Після цього тести слід запустити, зафіксувати результати та оцінити, чи функції працюють так, як очікувалося.

Програма 

```python
    
def add(a, b):
    return a + b

def subtract(a, b):
    return a - b

def multiply(a, b):
    return a * b

def divide(a, b):
    if b == 0:
        raise ValueError("divide on 0!")
    return a / b

```

Тести

```python
    
class Tests(unittest.TestCase):
    
    def test_add(self):
        self.assertEqual(add(14, 32), 46)
        self.assertEqual(add(-11, 11), 0)
        
    def test_subtract(self):
        self.assertEqual(subtract(10, 6), 4)
        self.assertEqual(subtract(0, 5), -5)
        
    def test_multiply(self):
        self.assertEqual(multiply(5, 5), 25)
        self.assertEqual(multiply(-2, 3), -6)
        
    def test_divide(self):
        self.assertEqual(divide(100, 2), 50)
        self.assertRaises(ValueError, divide, 100, 0)

```
Тут ми використали методи з бібліотеки unittest:

assertEqual який перевіряє, чи результат функції збігається з очікуваним значенням.

assertRaises який перевіряє, чи викликається певна помилка (наприклад, ділення на нуль).
