## Мета

Мета роботи – ознайомитися із техніками тестування програмного коду та навчитися писати unit-тести на Python 3.
## Шаблони

- Unit-тести (модульні) – перевіряють окремі функції чи класи.
- Integration-тести (інтеграційні) – перевіряють взаємодію між компонентами.
- System-тести (системні) – перевіряють роботу всієї системи в цілому.
- Acceptance-тести (приймальні) – перевіряють відповідність системи вимогам користувача.

## Хід роботи

- Ознайомився з видами тестів та пірамідою тестування.
- Вивчив модуль unittest, який використовується для написання unit-тестів у Python.
- Реалізував функцію add() для додавання двох чисел.
- Створив клас TestMath, що містить кілька тестів для різних випадків (додавання додатних, від’ємних чисел та нуля).
- Виконав тести за допомогою unittest.main() та переконався у правильності роботи функції.

## Висновок

У роботі розглянуто техніки тестування та піраміду тестування.
На прикладі unit-тесту з використанням модуля unittest показано, як перевіряти правильність роботи окремих функцій.
Тестування дозволяє своєчасно знаходити помилки, підвищує якість та надійність програмного забезпечення.
