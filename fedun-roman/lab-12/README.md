## Мета

Мета роботи – познайомитися з групою твірних шаблонів проєктування, розглянути їх призначення та реалізацію на прикладі.

## Хід роботи

- Ознайомився з теоретичними основами структурних шаблонів проєктування.
- Вивчив призначення шаблону Decorator, який дозволяє розширювати функціональність об’єктів без зміни їхнього коду.
- Реалізував приклад на Python:
- Створив базовий клас Product з методом get_price().
- Створив клас DiscountDecorator, який приймає об’єкт Product і змінює його поведінку, додаючи знижку.
- Перевірив роботу програми: викликав метод get_price() для звичайного продукту та для продукту з декоратором.

## Висновок

Factory Method дозволяє створювати об’єкти без прямої залежності від класів. Реалізація на Python показала гнучкість шаблону. UML-діаграма допомогла краще зрозуміти структуру проєкту.
