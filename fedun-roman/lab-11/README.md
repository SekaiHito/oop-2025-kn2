## Мета

Мета роботи – познайомитися з групою твірних шаблонів проєктування, розглянути їх призначення та реалізацію на прикладі.

## Хід роботи

- Ознайомився з теоретичними основами твірних шаблонів проєктування, зокрема з шаблоном Factory Method.
- Розглянув призначення шаблону: інкапсуляція процесу створення об’єктів, зменшення залежності від конкретних класів.
- Реалізував приклад на мові Python:
- Створено абстрактний клас Product з методом get_info().
- Створено два підкласи Book та Laptop, які реалізують метод get_info() відповідно.
- Створено клас ProductFactory, який містить метод create(type) для створення об’єктів залежно від типу.
- Перевірив роботу фабрики, створивши об’єкт типу Book та викликавши метод get_info().
- Побудував UML-діаграму, яка відображає структуру класів та їх взаємозв’язки.

## Висновок

Factory Method дозволяє створювати об’єкти без прямої залежності від класів. Реалізація на Python показала гнучкість шаблону. UML-діаграма допомогла краще зрозуміти структуру проєкту.
