## Хід роботи

1. Дати загальний опис принципів рефакторингу.
Рефакторинг — це дисциплінований процес внесення змін до внутрішньої структури програмного коду без зміни його зовнішньої поведінки. Його мета — покращити читабельність, ремонтопридатність та розширюваність коду.
2. Ознайомитися із основними техніками рефакторингу. Детально описати одну із технік.
Принципи Рефакторингу 
1. Переміщення (Moving)
Ці техніки стосуються зміни розташування коду:

1) Виділення Методу (Extract Method): Перетворення фрагмента коду всередині методу, що виконує певне завдання, на новий, окремий метод. (Найважливіша техніка)

2) Виділення Класу (Extract Class): Переміщення частини відповідальності та відповідних полів/методів до нового класу.

3) Переміщення Методу (Move Method): Перенесення методу з одного класу до іншого, де він використовує більше даних цього класу.

2. Спрощення (Simplifying)
Ці техніки зменшують складність і покращують читабельність:

Перейменування (Rename): Зміна назви методу, класу, поля чи змінної для кращого відображення їхньої мети.

Заміна Магічного Числа Константою (Replace Magic Number with Symbolic Constant): Заміна необґрунтованих числових значень іменованими константами.

Заміна Умовного Оператора Поліморфізмом (Replace Conditional with Polymorphism): Перетворення великого оператора switch/if-else на поліморфну поведінку, використовуючи наслідування або інтерфейси.

Видалення Дублюючого Коду (Remove Duplicated Code): Усунення однакових або дуже схожих блоків коду.
3. Композиція (Composing)
Ці техніки стосуються роботи зі структурою методів і класів:

Вбудовування Методу (Inline Method): Переміщення тіла методу у місце його виклику та видалення самого методу (протилежність Виділенню Методу).

Вбудовування Тимчасової Змінної (Inline Temp): Заміна тимчасової змінної виразом, яким вона ініціалізується.

Представлення [Об'єкта Параметра](refactoring.py) (Introduce Parameter Object): Якщо метод приймає багато пов'язаних параметрів, їх об'єднують в один об'єкт-параметр.

Детально: Виділення Методу (Extract Method)
Мета: Зменшити Довгі Методи (Long Methods) та підвищити читабельність.
Як: Виділити фрагмент коду, що виконує одну чітку дію, і помістити його в новий метод із пояснювальною назвою.
Результат: Основний метод стає змістом, що викликає допоміжні, легко зрозумілі методи.

3. Познайомитися із поняттям “запахів коду”. Дати детальний опис одного із
“запахів”.

"Запахи Коду" (Code Smells) — це термін, який використовується для позначення симптомів або показників у програмному коді, які свідчать про глибинні проблеми у його дизайні та структурі. Вони не є помилками (багами) в роботі програми, але ускладнюють її розуміння, модифікацію та підтримку.

Наявність "запаху" є сигналом для розробника, що код, ймовірно, потребує рефакторингу.
