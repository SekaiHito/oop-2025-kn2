## **Хід роботи**

1. **Реалізував патерн Adapter.**  
   Дає змогу класам із несумісними інтерфейсами працювати разом.
   ```py
# Старий інтерфейс
class OldPrinter:
    def print_text(self, text):
        print(f"Old printer: {text}")

# Новий інтерфейс, який нам потрібен
class NewPrinterInterface:
    def display(self, message):
        pass

# Адаптер
class PrinterAdapter(NewPrinterInterface):
    def __init__(self, old_printer):
        self.old_printer = old_printer

    def display(self, message):
        self.old_printer.print_text(message)

# Використання
old = OldPrinter()
adapter = PrinterAdapter(old)
adapter.display("Привіт, адаптер!")

   ```

2. **Структурні патерни (Structural).**  
Призначення: описують способи об'єднання об'єктів у структури та адаптацію інтерфейсів.

3. **Приклад: Adapter (Адаптер).**  
   Спрощує роботу з комплексною системою, надаючи єдиний інтерфейс.
Ідея:
дозволяє підключати несумісні компоненти шляхом перетворення інтерфейсу одного класу на інший.

Де використовується:
коли стара бібліотека не підходить під новий інтерфейс;
інтеграція сторонніх систем;
перехід зі старого API на нове.

Плюси:
дозволяє повторно використовувати старий код;
мінімізує зміни у програмі.


4. **Результат виконання програми:**
   ```
old = OldPrinter()               # Є старий принтер (старий інтерфейс)
adapter = PrinterAdapter(old)    # Створюємо адаптер, що робить його "новим"
adapter.display("Привіт, адаптер!")  # Викликаємо новий метод, який всередині використовує старий

Old printer: Привіт, адаптер!
