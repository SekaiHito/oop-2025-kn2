Хід роботи
1. Теоретичний опис поведінкової групи шаблонів. Поведінкові шаблони (Behavioral Patterns) — це категорія патернів проєктування, які вирішують завдання ефективної взаємодії між компонентами системи та розподілу обов'язків. На відміну від структурних патернів, що описують, як побудувати об'єкти, поведінкові описують, як об'єкти "спілкуються" один з одним, фокусуючись на алгоритмах та передачі відповідальності.

2. Реалізація патерну "Посередник" (Mediator). У ході лабораторної роботи було проаналізовано патерн "Посередник". Це шаблон, який дозволяє зменшити зв'язаність (coupling) безлічі класів між собою, переміщуючи ці зв'язки в один клас-посередник. Було розроблено програмний код мовою Python, який демонструє:

Створення абстрактного інтерфейсу Mediator.

Реалізацію конкретного посередника ConcreteMediator, що керує потоком виконання.

Взаємодію незалежних компонентів (Component1, Component2) через централізований хаб.

Також було побудовано діаграму класів для візуалізації взаємодії компонентів із посередником.
