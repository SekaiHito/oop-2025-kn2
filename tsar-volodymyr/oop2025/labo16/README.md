Мета: Вивчення методологій забезпечення якості програмного забезпечення, ознайомлення з рівнями тестування та набуття практичних навичок написання модульних тестів (Unit Tests) мовою Python.

Хід роботи
1. Теоретичний аналіз рівнів тестування. Було розглянуто основні рівні перевірки програмного забезпечення, що складають так звану "Піраміду тестування":

Unit Testing (Модульне): Перевірка найменших неподільних частин коду (функцій, методів, класів) в ізоляції.

Integration Testing (Інтеграційне): Перевірка коректності взаємодії між кількома модулями або компонентами системи.

System Testing (Системне): Тестування повністю інтегрованого продукту на відповідність технічним вимогам.

Acceptance Testing (Приймальне): Фінальна перевірка продукту на відповідність очікуванням кінцевого користувача (Business Requirements).

2. Робота з бібліотекою unittest. Для виконання практичної частини було обрано стандартну бібліотеку Python — unittest, яка базується на архітектурі xUnit.

3. Практична реалізація.

Розроблено тестову функцію add(a, b), що виконує арифметичне додавання двох чисел.

Створено клас TestMath, який успадковує unittest.TestCase.

Реалізовано набір тестових сценаріїв (Test Cases) для покриття різних умов:

Додавання додатних чисел.

Робота з від'ємними числами.

Перевірка граничних значень (робота з нулем).

Здійснено запуск тестів через unittest.main(). Усі тести пройдено успішно.

Висновки
У ході лабораторної роботи я опанував базові принципи автоматизованого тестування ПЗ. На практиці було застосовано фреймворк unittest для написання модульних тестів. Цей процес підтвердив, що Unit-тестування дозволяє виявляти помилки на ранніх етапах розробки, спрощує рефакторинг та підвищує загальну надійність програмного продукту.
