Мета: Ознайомитися з принципами побудови та використання групи структурних шаблонів проєктування.

Хід роботи
1. Теоретичний опис структурної групи шаблонів. Структурні шаблони (Structural Patterns) — це категорія патернів проєктування, що відповідають за побудову зручних ієрархій класів та об'єктів. Головна ідея полягає в тому, щоб з окремих простих частин складати більші структури, які залишаються гнучкими та ефективними. Ці шаблони допомагають вирішувати проблеми інтеграції різнорідних компонентів системи, оптимізувати зв'язки між сутностями та приховувати складність реалізації за простими інтерфейсами.

2. Реалізація патерну "Адаптер". У рамках роботи було розглянуто шаблон "Адаптер" (Adapter) — структурний патерн, який дозволяє об'єктам з несумісними інтерфейсами працювати разом. Було розроблено програмний код (adapte.py), у якому реалізовано:

Цільовий інтерфейс (Target) для нових платіжних процесорів.

Клас адаптера, що трансформує дані та виклики для роботи із застарілою системою (Legacy Code).

Демонстрацію роботи адаптера через композицію та спадкування.

Висновки
Під час виконання лабораторної роботи я ознайомився зі структурною групою шаблонів проєктування. Зокрема, детально вивчено патерн "Адаптер". На практиці реалізовано механізм адаптації несумісних інтерфейсів мовою Python, що дозволяє інтегрувати старий код у нову систему без його модифікації. Також було проаналізовано UML-діаграму взаємодії адаптера з клієнтом та адаптованим класом.
