## Мета

## Шаблони

- Adapter (Адаптер) — дозволяє узгодити два несумісні інтерфейси, перетворюючи один інтерфейс у інший.
- Facade (Фасад) — надає простий інтерфейс до складної системи, приховуючи її внутрішню реалізацію.
- Decorator (Декоратор) — додає нову функціональність об'єкту без зміни його класу, «обгортаючи» його.
- Proxy (Проксі) — контролює доступ до об’єкта, діючи як його замінник або посередник.
- Composite (Компонувальник) — дозволяє працювати з групою об’єктів як з одним цілим у вигляді деревоподібної структури.
- Bridge (Міст) — розділяє абстракцію та реалізацію, дозволяючи змінювати їх незалежно одна від одної.
- Flyweight (Легковаговик) — мінімізує використання пам’яті, розділяючи спільний стан між великою кількістю об'єктів.

## Хід роботи

- Ознайомився з основами структурних шаблонів проєктування
- Вивчив шаблон Adapter, який дозволяє робити несумісні інтерфейси сумісними.
- Реалізував приклад на Python:
- створив клас MediaPlayer з методом play();
- створив клас VideoPlayer з іншим методом play_video();
- створив адаптер VideoAdapter, який робить VideoPlayer сумісним з MediaPlayer;
- перевірив роботу програми, викликавши play() через адаптер для відеофайлу.

## Висновок

Factory Method дозволяє створювати об’єкти, не звертаючись безпосередньо до конкретних класів. Приклад на Python продемонстрував гнучкість цього шаблону, а UML-діаграма допомогла наочно уявити структуру проєкту.

      +----------------------+
      |     MediaPlayer      |
      |----------------------|
      | + play(audio)        |
      +----------^-----------+
                 |
                 |
     +------------------------+
     |     VideoAdapter       |
     |------------------------|
     | - video_player         |
     | + play(video)          |
     +-----------+------------+
                 |
      +----------------------+
      |     VideoPlayer      |
      |----------------------|
      | + play_video(video)  |
      +----------------------+
