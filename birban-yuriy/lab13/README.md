## Мета

Мета роботи – ознайомитися з групою поведінкових шаблонів проєктування та реалізувати один із них на практиці.

## Шаблони

Поведінкові
- Observer (Спостерігач) — один об’єкт повідомляє інших про зміни (наприклад, підписка на новини).
- Strategy (Стратегія) — дозволяє вибирати різні алгоритми виконання задачі (наприклад, різні способи оплати).
- Command (Команда) — перетворює дію в об’єкт, який можна зберігати, передавати чи скасовувати.
- State (Стан) — змінює поведінку об’єкта залежно від його внутрішнього стану.
- Iterator (Ітератор) — дає можливість проходити по елементах колекції без знання її внутрішньої структури.
- Mediator (Посередник) — координує взаємодію між кількома об’єктами.
- Chain of Responsibility (Ланцюг відповідальності) — запит проходить через ланцюг обробників, і кожен вирішує, чи обробляти його.


## Хід роботи

- Ознайомився з основами поведінкових шаблонів проєктування.
- Вивчив шаблон Observer, який забезпечує залежність "один-до-багатьох" між об’єктами.
- Створив клас NewsPublisher, який зберігає список підписників і повідомляє їх про новини.
- Створив клас Subscriber, який отримує оновлення через метод update.
- Перевірив роботу програми: після додавання новини (add_news) всі підписники отримали повідомлення.
- Створив UML-діяграму:

       +-----------------+
       | NewsPublisher   |
       |-----------------|
       | - subscribers   |
       | - news          |
       |-----------------|
       | + attach()      |
       | + detach()      |
       | + notify()      |
       | + add_news()    |
       +-----------------+
               ^
               |
               |
        +----------------+
        |   Subscriber   |
        |----------------|
        | - name         |
        |----------------|
        | + update(news) |
        +----------------+

## Висновок

У роботі розглянуто поведінкові шаблони та реалізовано приклад Observer. Він забезпечує механізм підписки, де зміна стану одного об’єкта автоматично повідомляє всі залежні. Реалізація на Python показала гнучкість цього підходу та спростила взаємодію між компонентами.
